# Bitvoya MCP v0.2.0

Public launch of Bitvoya MCP for luxury hotel discovery, rate comparison, quote preparation, and secure checkout handoff for AI travel agents.

Hosted endpoint:

- `https://bitvoya.com/api/mcp`

Agent key flow:

- `https://bitvoya.com/dashboard/agent-keys`

## Highlights

- public hosted remote MCP over Streamable HTTP
- registry-ready metadata in [server.json](../../../server.json)
- client setup guides for Cherry Studio, Cursor, Windsurf, Claude Code, GitHub Copilot CLI, and Goose
- secure checkout handoff model so card entry and payment stay on Bitvoya-owned surfaces
- Apache-2.0 licensing for the repository code and docs

## What Agents Can Do

- search luxury hotels and compare premium options
- inspect hotel detail, room, and rate data
- surface structured benefits and promotions
- prepare quotes and create booking intents
- poll booking and handoff state after traveler launch

Core tools:

- `search_hotels`
- `get_hotel_detail`
- `get_hotel_rooms`
- `compare_hotels`
- `compare_rates`
- `prepare_booking_quote`
- `create_booking_intent`
- `get_booking_state`

## Transport And Security

This release hardens the public remote MCP path for broader client compatibility:

- `GET`, `POST`, `DELETE`, `OPTIONS` support on the remote transport
- bearer auth enforced on remote `GET`
- Origin allowlisting and Host validation
- DNS rebinding protection support
- CORS and MCP session headers forwarded and exposed correctly

## Recommended Models

For best tool-call reliability, Bitvoya recommends higher-capability models such as:

- `Claude 4.6`
- `GPT-5.4`

Smaller models may still connect, but they are more likely to skip tools, flatten pricing semantics, or mishandle quote and state transitions.

## Notes

- normal users should connect to the hosted MCP instead of self-hosting
- MCP access uses revocable Bitvoya agent keys, not website passwords
- the open-source license does not grant access to Bitvoya production infrastructure, live inventory feeds, or private service credentials
